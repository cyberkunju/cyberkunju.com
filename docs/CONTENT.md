# Content authoring

How to add and edit the content that appears on the site.

## 1. Projects

Projects live as Markdown files in `src/content/projects/`. Each file is one
project. The filename (minus `.md`) becomes the URL slug:
`src/content/projects/my-thing.md` → `/projects/my-thing`.

### Frontmatter schema

Validated at build time by `src/content.config.ts` (Zod). Invalid frontmatter
**fails the build**.

```yaml
---
title: Project Name            # required, 1–80 chars
summary: One-line description. # required, 1–200 chars (shown on cards/lists)
description: Longer blurb.      # optional
date: 2026-06-01               # required (ISO date); used for sorting
updated: 2026-07-01            # optional
tags: ['rust', 'systems']      # optional string[]; default []
role: Lead engineer            # optional (shown as an eyebrow)
stack: ['Rust', 'Axum']        # optional string[]; first 4 shown as tags on cards
links:                         # optional; all URLs validated
  repo: https://github.com/...
  live: https://...
  writeup: https://...
cover: ./cover.png             # optional image (processed by astro:assets)
coverAlt: Alt text.            # optional (use when cover is set)
featured: true                 # optional; true → appears on the home page
draft: false                   # optional; true → excluded from the site
order: 1                       # optional int; lower sorts first
---

Markdown body goes here. Headings, lists, code blocks (Shiki dual-theme), etc.
```

### How projects are selected & sorted

- **Home page** (`src/pages/index.astro`): `getCollection('projects', p => !p.data.draft)`
  filtered to `featured`, sorted by `order` (asc), then `date` (desc).
- **All projects** (`src/pages/projects/index.astro`): lists non-draft projects.
- **Detail page** (`src/pages/projects/[...slug].astro`): renders one project's
  body; can pass `jsonLdExtra` for richer structured data.

### Images / covers

- Put the cover image next to the Markdown file and reference it relatively
  (`cover: ./cover.png`). `astro:assets` + Sharp emit responsive AVIF with no
  layout shift.
- Always set `coverAlt` for accessibility (a11y score is enforced at 1.0).

## 2. Pages

Static pages are `.astro` files in `src/pages/`:

- `index.astro` — hero (name, job title, description, CTAs) + selected work.
- `about.astro` — the About content (edit the markup directly).
- `projects/index.astro` — the work index.
- `404.astro` — not-found page.
- `design.astro` — internal style guide; `noindex` and excluded from the
  sitemap. Handy reference for tokens/components.

Every page wraps `BaseLayout` and can pass props:

```astro
<BaseLayout
  title="About"                 // becomes "About — Navaneeth K"
  description="..."             // overrides default meta description
  noindex={false}
  type="website"                // or "article" for project pages
  jsonLdExtra={[ ... ]}         // optional extra schema.org objects
>
  ... page content ...
</BaseLayout>
```

## 3. The Open Graph card

`public/og-default.png` (1200×630) is the default social-share image. It's
generated by `scripts/generate-og.mjs` (Sharp). To change it:

1. Edit the `NAME` / `TITLE` / `DOMAIN` constants (and layout) in the script.
2. Run `bun run og`.
3. Commit the regenerated `public/og-default.png`.

Per-page images can be passed via the `image` prop to `BaseLayout`/`SEO`.

## 4. Writing style / links

- Body Markdown is styled by `.prose` (`src/styles/prose.css`): readable
  measure, underlined links that turn accent on hover, styled code blocks.
- External links that shouldn't be prerendered should carry `data-no-prerender`
  (e.g. `mailto:` and off-site links already do in the templates).
